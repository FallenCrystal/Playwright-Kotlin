# Playwright-Kotlin

A Kotlin/JVM [Playwright](https://playwright.dev/) browser automation client with native coroutine support.

All browser operations are `suspend` functions, naturally compatible with Kotlin coroutines. The API style stays consistent with the official Playwright API.

## ⚠ Notice

The code for this project is mostly generated by Claude Code.

The official `playwright-java` can only be called on a single thread. And all calls are blocking.

I do not plan to migrate my project to TypeScript. This is a compromise solution.

Its API is also incomplete. It may also have memory leak issues.

I don't constrain the coding style. You might see repeated code and so on.

I do not provide any support for this project. You can fork it and make changes.

## How It Works

Playwright-Kotlin uses a **two-process architecture**:

```
Kotlin Client (Netty TCP) ←── NDJSON Protocol ──→ Node.js Server (Real Playwright)
```

The Kotlin side sends JSON requests to a TypeScript Server over TCP. The Server invokes the real Playwright API to perform operations and returns results. All remote objects are identified and tracked via GUIDs.

## Prerequisites

- JDK 21+
- Node.js 18+, npm
- Playwright browsers (at least one installed)

## Quick Start

### 1. Build the Server

```bash
cd server
npm install
npm run build
npx playwright install chromium   # Install Chromium browser
```

### 2. Build the Kotlin Client

```bash
./gradlew :playwright-kotlin:build
```

### 3. Usage Example

```kotlin
import io.playwright.kotlin.*
import io.playwright.kotlin.options.LaunchOptions
import kotlinx.coroutines.runBlocking

fun main() = runBlocking {
    val pw = Playwright.create()
    try {
        val browser = pw.chromium.launch(LaunchOptions(headless = true))
        val page = browser.newPage()

        // Navigation
        val response = page.goto("https://example.com")
        println("Status: ${response?.status()}")

        // Get page title
        println("Title: ${page.title()}")

        // Locate elements with Locator
        val h1 = page.locator("h1")
        println("H1 text: ${h1.textContent()}")

        // Form filling
        // page.fill("#username", "admin")
        // page.click("#submit")

        // Screenshot
        val screenshot = page.screenshot()
        println("Screenshot size: ${screenshot.size} bytes")

        // Execute JavaScript
        val title = page.evaluate("document.title")
        println("JS result: $title")

        browser.close()
    } finally {
        pw.close()
    }
}
```

## API Overview

### Entry Point

| Class | Description |
|---|---|
| `Playwright` | Top-level entry point, created via `Playwright.create()`, provides three browser types: `chromium`, `firefox`, `webkit` |
| `PlaywrightConfig` | Configuration options: `host`, `port`, `serverPath`, `nodeExecutable`, `launchTimeout` |

### Browser Management

| Class | Key Methods |
|---|---|
| `BrowserType` | `launch(options)` — Launch a browser instance |
| `Browser` | `newContext()`, `newPage()`, `close()`, `contexts()`, `isConnected()` |
| `BrowserContext` | `newPage()`, `pages()`, `cookies()`, `addCookies()`, `clearCookies()`, `close()` |

### Page Operations

| Class | Key Methods |
|---|---|
| `Page` | `goto()`, `locator()`, `click()`, `fill()`, `type()`, `press()`, `screenshot()`, `evaluate()`, `content()`, `title()`, `url()`, `close()` |
| `Frame` | Navigation and manipulation methods similar to Page |

### Element Operations

| Class | Key Methods |
|---|---|
| `Locator` | `click()`, `fill()`, `textContent()`, `isVisible()`, `count()`, `locator()` (chained locating), `first()`, `nth()`, `last()` |
| `ElementHandle` | `click()`, `fill()`, `textContent()`, `getAttribute()`, `boundingBox()`, `screenshot()` |

### Network

| Class | Key Methods |
|---|---|
| `Response` | `status()`, `ok()`, `url()`, `headers()`, `body()`, `text()`, `json()` |

## Runtime Modules

Playwright-Kotlin splits runtimes into independent Gradle modules, automatically discovered via SPI (`ServiceLoader`). Users only need to add the corresponding `runtimeOnly` dependency — zero configuration required to get started.

### Available Runtimes

| Module | Maven Coordinates | Description |
|--------|------------------|-------------|
| Node.js Runtime | `io.playwright:playwright-kotlin-runtime-node` | Bundles `server-bundle.js`, requires Node.js installed on the system |
| Linux x64 Prebuilt | `io.playwright:playwright-kotlin-runtime-prebuild-linux-x64` | Embeds Linux x64 SEA binary, no Node.js required |
| Windows x64 Prebuilt | `io.playwright:playwright-kotlin-runtime-prebuild-win-x64` | Embeds Windows x64 SEA binary, no Node.js required |
| macOS x64 Prebuilt | `io.playwright:playwright-kotlin-runtime-prebuild-macos-x64` | Embeds macOS x64 SEA binary, no Node.js required |

### Dependency Configuration Example

```kotlin
dependencies {
    implementation("io.playwright:playwright-kotlin:0.1.0")

    // Option 1: Node.js runtime (requires Node.js environment)
    runtimeOnly("io.playwright:playwright-kotlin-runtime-node:0.1.0")

    // Option 2: Prebuilt binary (no Node.js needed, choose one for your platform)
    // runtimeOnly("io.playwright:playwright-kotlin-runtime-prebuild-linux-x64:0.1.0")
    // runtimeOnly("io.playwright:playwright-kotlin-runtime-prebuild-win-x64:0.1.0")
    // runtimeOnly("io.playwright:playwright-kotlin-runtime-prebuild-macos-x64:0.1.0")
}
```

### Priority Rules

When multiple runtimes are present on the classpath, the best one is selected automatically by priority:

1. **Prebuilt binary** (priority=100) — Used first if the platform matches and the resource exists
2. **Node.js Bundle** (priority=50) — Requires Node.js, but works cross-platform
3. **Local Fallback** (lowest priority) — Searches for the `server/dist/index.js` development script

## Connection Modes

Playwright-Kotlin supports multiple connection modes:

**Auto mode** (default) — Automatically discovers a runtime via SPI and starts the Server:

```kotlin
val pw = Playwright.create()  // Automatically selects the best runtime
```

**Remote mode** — Connects to an already running Server:

```kotlin
val pw = Playwright.create(PlaywrightConfig(host = "127.0.0.1", port = 3000))
```

**Custom Server path** — Manually specify the Server script:

```kotlin
val pw = Playwright.create(PlaywrightConfig(serverPath = "/path/to/server/dist/index.js"))
```

## Protocol Format

The client and server communicate using NDJSON (newline-delimited JSON):

```
Request:  {"id":1, "guid":"page-abc", "method":"goto", "params":{"url":"..."}}
Response: {"id":1, "result":{"guid":"response-xyz", "type":"Response"}}
Error:    {"id":1, "error":{"name":"TimeoutError", "message":"..."}}
Event:    {"guid":"page-abc", "method":"__event__", "params":{"type":"close"}}
```
